Coordinate System
If panos are rotated differently per station, the quat encoding is wrong. Try: conjugate first, then permutations, then component negations. Watch the colored axis lines on each marker — when those align with the actual scanner orientations in the scene, the encoding is right.
Station Transform
Apply scale + offset to station coordinates to align them with the PLY. Use auto-fit to guess.
Quaternion Convention
If pano rotation varies per station (not consistent error), try these.
"Component remap" reassigns which JSON axis becomes which Three.js axis. Often needed when the source data is in Z-up but you're applying it in Y-up space.
Pano Sphere
Aligns the sphere's pole with the scanner's vertical axis. Try options until the black nadir is on the floor.
Alignment Tools
For dialing in seam offset: set test alpha to ~0.5 and look for features (corners, doorways) that should overlap between the pano and the point cloud. Or hold ALT to temporarily make the sphere transparent. Once one station's seam is correct, all should be.
